# MealMind AI｜AI 外卖商家经营助手

> 面向外卖商家的 AI 经营诊断与运营决策助手，基于订单、评价与营销数据，帮助商家发现经营异常、分析原因、生成策略并辅助客服回复。

---

## 1. 项目亮点

MealMind AI 是一个面向 **本地生活 / 外卖商家经营场景** 的 AI 产品经理作品集项目。

本项目尝试将：

- 大语言模型 LLM
- Agent 工作流
- 评价文本分析
- 经营数据分析
- 营销策略生成
- 客服回复辅助

应用到外卖商家的真实经营问题中，形成从 **数据输入 → 异常识别 → 原因分析 → 策略生成 → 执行辅助 → 效果追踪** 的产品闭环。

---

## 2. 为什么做这个项目

外卖中小商家在日常经营中经常遇到以下问题：

| 商家问题 | 典型表现 |
|---|---|
| 看不懂经营数据 | 只知道订单少了，但不知道原因 |
| 差评归因困难 | 用户评价分散，难以快速判断问题集中在哪里 |
| 营销活动依赖经验 | 不知道该做满减、复购券还是套餐组合 |
| 客服回复效率低 | 差评和投诉回复耗时，且容易措辞不当 |
| 经营建议不够可执行 | 后台只展示数据，但缺少明确行动建议 |

因此，MealMind AI 的目标不是再做一个商家后台，而是做商家后台之上的 **AI 智能分析层**。

---

## 3. 产品定位

MealMind AI 的产品定位是：

```text
面向外卖中小商家的 AI 经营分析与运营决策助手
```

它不是普通聊天机器人，也不是单纯的数据看板，而是围绕外卖商家的核心经营问题，提供：

| 能力 | 说明 |
|---|---|
| 经营诊断 | 分析订单下降、转化下降、差评上升等异常 |
| 评价洞察 | 对用户评价进行情绪识别、关键词提取和问题归因 |
| 营销建议 | 生成复购券、满减、套餐、爆品等运营策略 |
| 客服辅助 | 自动生成差评回复、投诉安抚和补偿引导话术 |
| Agent 协作 | 多 Agent 分工完成诊断、归因、策略和回复生成 |

## 4. 核心业务闭环

```text
经营数据输入
↓
异常指标识别
↓
AI 多维归因分析
↓
运营策略生成
↓
客服与营销辅助执行
↓
效果追踪与迭代优化
```

对应到产品中：

| 业务环节 | 产品模块 | 解决的问题 |
|---|---|---|
| 数据输入 | 数据接入层 | 获取订单、评价、营销、客服等基础数据 |
| 异常识别 | 经营看板 | 帮助商家发现订单下降、差评上升等问题 |
| 原因分析 | AI 经营诊断 | 分析异常背后的可能原因 |
| 策略生成 | 营销策略 Agent | 生成可执行的运营建议 |
| 执行辅助 | 客服话术 Agent | 生成评价回复和投诉处理话术 |
| 效果追踪 | 数据指标体系 | 判断建议执行后是否有效 |

## 6. Agent 设计

MealMind AI 设计了 4 个核心 Agent：

| Agent | 主要职责 | 输出结果 |
|---|---|---|
| 数据分析 Agent | 分析订单、流量、转化率、客单价、复购率等经营指标 | 异常指标、关键证据、初步原因 |
| 评价洞察 Agent | 分析用户评价、差评原因、高频关键词和问题菜品 | 差评分类、关键词、问题菜品 |
| 营销策略 Agent | 生成优惠券、套餐、爆品和复购策略 | 营销建议、适用用户、风险提示 |
| 客服话术 Agent | 生成评价回复、投诉安抚和补偿引导话术 | 标准版、诚恳版、补偿引导版回复 |

Agent 协作流程：

```text
商家提出问题
↓
系统识别问题类型
↓
数据分析 Agent 判断经营异常
↓
评价洞察 Agent 分析用户反馈
↓
营销策略 Agent 生成运营建议
↓
客服话术 Agent 生成回复模板
↓
系统汇总为经营诊断报告
```

## 7. 典型使用场景

### 场景一：订单下降诊断

商家提问：

```text
为什么我这周订单下降了？
```

系统分析：

```text
订单量变化
↓
曝光量变化
↓
转化率变化
↓
差评率变化
↓
营销活动变化
↓
输出诊断报告
```

示例输出：

```text
本周订单量较上周下降 18%。主要原因可能是曝光量下降、转化率下降和包装类差评上升共同导致。

关键证据：
1. 曝光量下降 12%，说明进入店铺的潜在用户减少。
2. 下单转化率下降 6%，说明用户进店后下单意愿降低。
3. 包装类差评占近期差评的 42%，主要集中在汤类菜品。

建议：
1. 优先优化汤类菜品密封包装。
2. 在商品页增加“汤粉分装”“密封包装”说明。
3. 对近 30 天购买过汤类菜品的老客发放复购券。
```

---

### 场景二：差评归因分析

商家提问：

```text
最近为什么差评变多了？
```

系统输出：

```text
差评主要集中在包装问题和配送问题。

高频关键词：
- 汤洒了
- 包装差
- 配送慢
- 等太久

问题菜品：
- 番茄肥牛汤
- 牛肉汤饭
- 酸辣粉

建议：
优先更换汤类菜品包装，并针对包装类差评生成诚恳回复话术。
```

## 12. 数据指标体系

MealMind AI 的指标体系分为四类：

| 指标类型 | 说明 |
|---|---|
| 产品使用指标 | 判断商家是否愿意使用 AI 功能 |
| AI 质量指标 | 判断 AI 输出是否准确、可信、可执行 |
| 业务效果指标 | 判断建议执行后经营是否改善 |
| 风险控制指标 | 判断 AI 是否存在幻觉、错误归因或不当回复 |

北极星指标：

```text
AI 经营建议采纳率 = 被商家采纳的 AI 建议数量 / AI 生成的经营建议总数量
```

选择该指标的原因：

- 反映商家是否信任 AI；
- 反映 AI 建议是否可执行；
- 比单纯点击率更接近真实产品价值；
- 可以进一步关联订单量、差评率、复购率等经营结果。

## 14. 项目亮点总结

### 1. 贴近真实业务场景

项目聚焦本地生活平台中的外卖商家经营，不是泛泛设计一个通用 AI 助手。

### 2. 具备完整产品链路

从用户痛点、竞品分析、功能架构、PRD、Agent 工作流到数据指标体系，形成完整 AI 产品设计链路。

### 3. 突出 AI 产品能力

项目中体现了 LLM、Agent、Prompt、RAG、AI 质量指标和风险控制等 AI 产品设计要素。

### 4. 有数据和 Demo 支撑

项目不仅有文档，还构建了订单、评价、营销 Mock 数据，并使用 Streamlit 搭建了基础演示页面。

### 5. 可用于 AI 产品经理作品集

适合作为投递 AI 产品经理实习岗位时的核心项目，尤其适用于本地生活、商家经营、AIGC 应用、AI Agent、智能客服等方向。
