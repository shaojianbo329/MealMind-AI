# MealMind AI｜AI 外卖商家经营助手

## 项目简介

MealMind AI 是一款面向外卖商家的智能经营分析与运营决策助手。

本项目基于外卖商家的订单数据、评价数据和营销活动数据，设计了一套从经营异常发现、原因分析、策略生成到客服回复辅助的 AI 产品方案。

项目重点展示 AI 产品经理所需的能力，包括：

- 业务场景理解
- 用户痛点分析
- 竞品分析
- 产品功能架构设计
- PRD 撰写
- Agent 工作流设计
- 数据指标体系设计
- Mock 数据构建
- Streamlit Demo 原型实现

---

## 产品定位

MealMind AI 不是一个普通聊天机器人，也不是单纯的数据看板，而是面向外卖商家的 AI 经营助手。

它希望帮助商家解决以下问题：

| 商家问题 | MealMind AI 的解决方式 |
|---|---|
| 看不懂经营数据 | 将订单、评价、营销数据转化为自然语言诊断 |
| 不知道订单为什么下降 | 通过 AI 分析曝光、转化、差评、活动等多维因素 |
| 不知道差评集中在哪里 | 对评价进行情绪识别、关键词提取和问题归因 |
| 不会设计营销活动 | 根据经营目标生成优惠券、套餐和复购策略 |
| 回复差评效率低 | 自动生成评价回复和投诉安抚话术 |

---

## 核心业务闭环

```text
经营数据输入
↓
异常指标识别
↓
AI 多维归因分析
↓
运营策略生成
↓
客服与营销辅助执行
↓
效果追踪与迭代优化
```

---

## 核心功能

| 功能模块 | 功能说明 |
|---|---|
| 经营看板 | 展示订单量、营业额、客单价、评分、配送时长、差评率等核心指标 |
| AI 经营诊断 | 分析订单下降、转化率下降、差评上升等经营异常 |
| 评价洞察分析 | 对用户评价进行情绪识别、问题分类和关键词提取 |
| 营销策略生成 | 根据经营目标生成新客券、复购券、套餐和爆品策略 |
| 客服回复辅助 | 根据差评和投诉内容生成标准化回复话术 |
| Agent 工作流 | 通过多个 Agent 协作完成经营诊断和策略生成 |

## Agent 设计

MealMind AI 设计了 4 个核心 Agent：

| Agent | 主要职责 |
|---|---|
| 数据分析 Agent | 分析订单、流量、转化率、客单价、复购率等经营指标 |
| 评价洞察 Agent | 分析用户评价、差评原因、高频关键词和问题菜品 |
| 营销策略 Agent | 生成优惠券、套餐、爆品和复购策略 |
| 客服话术 Agent | 生成评价回复、投诉安抚和补偿引导话术 |

Agent 协作流程：

```text
商家提出问题
↓
系统识别问题类型
↓
数据分析 Agent 判断经营异常
↓
评价洞察 Agent 分析用户反馈
↓
营销策略 Agent 生成运营建议
↓
客服话术 Agent 生成回复模板
↓
系统汇总为经营诊断报告
```

## Demo 运行方式

进入项目根目录后执行：

```bash
pip install -r demo/requirements.txt
streamlit run demo/app.py
```

Demo 包含以下模块：

1. 经营看板
2. 差评归因分析
3. 营销活动分析
4. AI 经营诊断示例

当前 Demo 使用模拟数据和规则化示例，后续可以接入大语言模型 API，实现动态诊断和策略生成。

## 数据指标体系

MealMind AI 的指标体系分为四类：

| 指标类型 | 说明 |
|---|---|
| 产品使用指标 | 判断商家是否愿意使用 AI 功能 |
| AI 质量指标 | 判断 AI 输出是否准确、可信、可执行 |
| 业务效果指标 | 判断建议执行后经营是否改善 |
| 风险控制指标 | 判断 AI 是否存在幻觉、错误归因或不当回复 |

北极星指标：

```text
AI 经营建议采纳率 = 被商家采纳的 AI 建议数量 / AI 生成的经营建议总数量
```

## 项目亮点

1. **贴近真实业务场景**  
   项目聚焦外卖商家经营，不是泛泛而谈的 AI 助手。

   从用户痛点、竞品分析、功能架构、PRD 到指标体系均有完整设计。

3. **突出 AI 产品能力**  
   引入 RAG、Agent、Prompt、AI 质量指标和风险控制思路。

4. **有可运行 Demo**  
   使用 Streamlit 和模拟数据搭建了基础演示页面。

5. **适合 AI 产品经理作品集展示**  
   能体现产品思维、AI 理解、数据意识和落地表达能力。
