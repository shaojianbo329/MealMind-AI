# 第 3 页｜项目背景：为什么做这个项目

## 背景问题

外卖中小商家每天都要面对订单量、营业额、客单价、评分、差评率、营销活动效果等多个指标。

但很多商家没有专业数据分析能力，常常只能看到结果，却难以理解原因。

例如：

```text
订单下降了
↓
不知道是曝光下降、转化下降、差评增加，还是营销活动失效
↓
不知道下一步应该优化菜品、调整活动，还是回复差评
```

## 产品机会

现有商家后台更偏向展示数据，而商家真正需要的是：

```text
发生了什么？
为什么发生？
我应该先做什么？
做完以后看什么指标？
```

MealMind AI 的机会点就在于：  
将订单、评价和营销数据转化为商家能理解、能执行、能复盘的经营建议。

## 目标用户

| 用户类型 | 典型特征 | 核心需求 |
|---|---|---|
| 中小外卖商家 | 缺少专业运营人员，依赖经验经营 | 快速发现问题并获得建议 |

## 核心痛点

| 痛点 | 具体表现 |
|---|---|
| 看不懂经营数据 | 只知道订单下降，不知道原因 |
| 差评归因困难 | 用户评价分散，人工查看效率低 |
| 营销依赖经验 | 不知道该做满减、复购券还是套餐 |

| 建议不够可执行 | 后台展示指标，但缺少明确行动建议 |

| 通用 AI 助手 | 对话和生成能力强 | 缺少业务数据和平台上下文 |

```text
商家后台数据
+
大语言模型解释能力
+
Agent 任务拆解
+
经营建议生成
+
客服回复辅助
```

传统后台告诉商家：

```text

```

MealMind AI 应该告诉商家：

```text

建议优先优化汤类菜品包装，并针对老客发放复购券。
```

# 第 7 页｜产品定位与业务闭环

## 产品定位

MealMind AI 是面向外卖中小商家的 AI 经营分析与运营决策助手。

它不是普通聊天机器人，也不是单纯数据看板，而是商家后台之上的 AI 智能分析层。

## 核心业务闭环

```text
经营数据输入
↓
异常指标识别
↓
AI 多维归因分析
↓
运营策略生成
↓
客服与营销辅助执行
↓
效果追踪与迭代优化
```

## 产品价值

| 环节 | 产品价值 |
|---|---|
| 数据输入 | 汇总订单、评价、营销、客服等数据 |
| 异常识别 | 帮助商家快速发现订单下降、差评上升等问题 |
| 原因分析 | 用 AI 解释经营异常背后的可能原因 |
| 策略生成 | 给出可执行的运营建议 |
| 执行辅助 | 生成客服回复和营销动作建议 |
| 效果追踪 | 判断建议执行后是否有效 |

# 第 9 页｜Agent 工作流设计

## 设计思路

外卖商家的经营问题通常不是单一原因导致的。

例如“订单下降”可能同时涉及：

```text
曝光量下降
转化率下降
差评增加
包装体验变差
营销活动减少
配送时长增加
```

因此，MealMind AI 设计了多个 Agent 分工协作，而不是只让一个通用 AI 给出空泛建议。

## 四个核心 Agent

| Agent | 主要职责 | 输出结果 |
|---|---|---|
| 数据分析 Agent | 分析订单、流量、转化率、客单价、复购率等指标 | 异常指标、关键证据、初步原因 |
| 评价洞察 Agent | 分析用户评价、差评原因、高频关键词和问题菜品 | 差评分类、关键词、问题菜品 |
| 营销策略 Agent | 生成优惠券、套餐、爆品和复购策略 | 营销建议、适用用户、风险提示 |
| 客服话术 Agent | 生成评价回复、投诉安抚和补偿引导话术 | 标准版、诚恳版、补偿引导版回复 |

## 协作流程

```text
商家提出问题
↓
系统识别问题类型
↓
数据分析 Agent 判断经营异常
↓
评价洞察 Agent 分析用户反馈
↓
营销策略 Agent 生成运营建议
↓
客服话术 Agent 生成回复模板
↓
系统汇总为经营诊断报告
```

# 第 10 页｜PRD 核心功能示例：AI 经营诊断

## 功能目标

帮助商家分析经营异常原因，并生成有证据支撑的诊断报告。

## 功能入口

| 入口 | 说明 |
|---|---|
| 经营看板异常指标 | 点击订单下降、差评上升等异常指标进入 |
| 首页 AI 提问框 | 商家直接输入经营问题 |
| 诊断历史记录 | 查看过去生成的诊断报告 |

## 输入内容

| 输入类型 | 示例 |
|---|---|
| 商家自然语言问题 | 为什么我这周订单下降了？ |

| 评价数据 | 评分、评价文本、关键词 |
| 营销数据 | 活动类型、优惠力度、活动时间 |
| 菜品数据 | 菜品销量、价格、退款率 |

## 输出内容

AI 诊断报告包括：

1. 异常现象说明
2. 关键数据变化
3. 可能原因排序
4. 支撑证据
5. 优化建议
6. 建议优先级
7. 后续观察指标

# 第 11 页｜AI 诊断报告示例

## 商家问题

```text
为什么我这周订单下降了？
```

## AI 输出示例

```text
本周订单量较上周下降 18%。

主要原因可能是曝光量下降、转化率下降和包装类差评上升共同导致。
```

## 关键证据

```text
1. 曝光量下降 12%，说明进入店铺的潜在用户减少。
2. 下单转化率下降 6%，说明用户进店后下单意愿降低。
3. 近 7 天“包装洒漏”相关差评明显增加，占差评总量的 42%。
```

## 优化建议

```text
1. 优先优化汤类菜品包装。
2. 对近 30 天购买过汤类菜品的用户发放复购券。
3. 在商品页增加“汤粉分装”“密封包装”说明。
```

## 后续观察指标

```text
订单量、转化率、包装类差评占比、复购券使用率、活动 ROI
```

# 第 12 页｜数据指标体系

## 指标设计目标

MealMind AI 的指标体系需要同时回答四个问题：

```text
商家是否愿意使用？
AI 输出是否可信？
建议是否被采纳？
执行后是否改善？
```

## 四类指标

| 指标类型 | 说明 |
|---|---|
| 产品使用指标 | 判断商家是否愿意使用 AI 功能 |
| AI 质量指标 | 判断 AI 输出是否准确、可信、可执行 |
| 业务效果指标 | 判断建议执行后经营是否改善 |
| 风险控制指标 | 判断 AI 是否存在幻觉、错误归因或不当回复 |

## 北极星指标

```text
AI 经营建议采纳率 = 被商家采纳的 AI 建议数量 / AI 生成的经营建议总数量
```

## 选择原因

- 反映商家是否信任 AI；
- 反映 AI 建议是否可执行；
- 比单纯点击率更接近真实产品价值；
- 可以关联订单量、差评率、复购率等业务结果。

1. AI 产品要先找到真实业务问题；
2. 大模型能力必须和业务数据结合；

4. AI 输出需要可解释、可执行、可评估；
5. 指标体系要同时关注使用、质量、业务和风险。
