## 03 项目背景：为什么做这个项目

### 页面核心

说明外卖商家为什么需要 AI 经营助手。

### 页面内容

外卖中小商家在日常经营中常常面临订单下降、差评增加、活动效果不佳、客服回复慢等问题。虽然平台后台会展示大量数据，但很多商家缺少专业运营和数据分析能力，难以判断指标变化背后的原因。

MealMind AI 的机会点在于：不只是展示数据，而是将复杂经营数据转化为商家能理解、能执行、能追踪的运营建议。

### 可视化建议

放一个流程图：

```text
订单下降
↓
不知道原因
↓
查看评价和营销数据成本高
↓
AI 自动分析
↓
输出原因和建议
```

## 04 用户痛点与目标用户

### 页面核心

展示目标用户和痛点。

### 目标用户

| 用户类型 | 典型特征 | 核心需求 |
|---|---|---|
| 中小外卖商家 | 缺少专业运营人员 | 快速发现问题并获得建议 |
| 门店运营人员 | 每天查看订单、评价和活动数据 | 提高分析和执行效率 |
| 连锁品牌区域经理 | 管理多家门店 | 快速识别异常门店 |
| 平台商家运营人员 | 负责商家扶持 | 提高商家服务效率 |

### 核心痛点

| 痛点 | 说明 |
|---|---|
| 看不懂经营数据 | 只知道订单下降，不知道原因 |
| 差评归因困难 | 用户评价分散，人工分析效率低 |
| 营销依赖经验 | 不知道该做满减、复购券还是套餐 |
| 客服回复效率低 | 回复差评耗时，且容易措辞不当 |
| 建议不可执行 | 后台展示指标，但缺少行动建议 |

## 06 竞品分析与机会点

### 页面核心

说明现有产品有什么不足，MealMind AI 的机会在哪里。

### 对比维度

| 产品类型 | 优势 | 不足 |
|---|---|---|
| 外卖商家后台 | 数据完整，业务链路闭环 | 分析门槛高，解释能力不足 |
| 本地生活商家工具 | 流量和营销能力强 | 对外卖经营诊断深度有限 |
| 通用 AI 助手 | 对话和生成能力强 | 缺少业务数据和平台上下文 |
| RAG 客服助手 | 知识问答准确性较高 | 偏客服，不够经营决策化 |

### 机会点

MealMind AI 的机会在于：

```text
商家后台数据
+
大语言模型解释能力
+
Agent 任务拆解
+
经营建议与客服辅助
```

形成一个面向外卖商家的 AI 决策辅助层。

## 10 PRD 核心页面

### 页面核心

展示你会写产品需求文档。

### 可以放的内容

选择一个核心功能：**AI 经营诊断**

| 项目 | 内容 |
|---|---|
| 功能目标 | 帮助商家分析经营异常原因，并生成有证据支撑的诊断报告 |
| 功能入口 | 经营看板异常指标、首页 AI 提问框、诊断历史 |
| 输入内容 | 商家问题、订单数据、评价数据、营销数据、菜品数据 |
| 输出内容 | 异常说明、关键数据、可能原因、优化建议、后续指标 |
| 异常情况 | 数据不足、评价过少、AI 生成失败、指标变化不明显 |

### 示例输出

```text
本周订单量较上周下降 18%。

关键证据：
1. 曝光量下降 12%；
2. 下单转化率下降 6%；
3. 包装类差评明显增加。

建议：
1. 优化汤类菜品包装；
2. 发放老客复购券；
3. 增加商品页包装说明。
```

## 11 数据指标体系

### 页面核心

展示你有数据意识和 AI 产品评估意识。

### 四类指标

| 指标类型 | 说明 |
|---|---|
| 产品使用指标 | 商家是否愿意使用 AI 功能 |
| AI 质量指标 | AI 输出是否准确、可信、可执行 |
| 业务效果指标 | 建议执行后经营是否改善 |
| 风险控制指标 | AI 是否存在幻觉、错误归因或不当回复 |

### 北极星指标

```text
AI 经营建议采纳率 = 被商家采纳的 AI 建议数量 / AI 生成的经营建议总数量
```

### 选择原因

- 反映商家是否信任 AI；
- 反映 AI 建议是否可执行；
- 比点击率更接近真实产品价值；
- 可以关联订单量、差评率、复购率等业务结果。

## 13 项目亮点与复盘

### 页面核心

总结这个项目为什么有价值。

### 项目亮点

| 亮点 | 说明 |
|---|---|
| 场景真实 | 聚焦外卖商家经营场景 |
| 链路完整 | 覆盖痛点、竞品、PRD、Agent、指标、Demo |
| AI 产品化 | 不是泛泛聊天，而是任务拆解和经营辅助 |
| 有数据支撑 | 构建订单、评价、营销 Mock 数据 |
| 可展示 | GitHub + Streamlit Demo + 作品集 PDF |

### 项目复盘

通过这个项目，我认识到：

1. AI 产品要先找到真实业务问题；
2. 大模型能力必须和业务数据结合；
3. Agent 设计要服务任务拆解，而不是堆概念；
4. AI 输出需要可解释、可执行、可评估；
5. 指标体系要同时关注使用、质量、业务和风险。
