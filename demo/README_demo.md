# Demo 使用说明

## 一、Demo 简介

本 Demo 是 MealMind AI 的最简可运行原型，用于展示 AI 外卖商家经营助手的核心产品逻辑。

Demo 基于模拟订单数据、评价数据和营销活动数据，展示以下功能：

1. 经营看板
2. 差评归因分析
3. 营销活动分析
4. AI 经营诊断示例

本 Demo 的目标不是实现完整商业系统，而是用于验证 MealMind AI 的产品思路、数据结构和核心交互流程。

---

## 三、运行环境

建议使用 Python 3.9 及以上版本。

需要安装的依赖：

```text
streamlit
pandas
```

---

## 四、本地运行方式

在项目根目录下执行：

```bash
pip install -r demo/requirements.txt
streamlit run demo/app.py
```

运行后，浏览器会自动打开本地页面。

## 五、Demo 功能说明

## 1. 经营看板

经营看板展示商家的基础经营指标，包括：

| 指标 | 说明 |
|---|---|
| 订单量 | 模拟周期内完成的订单数量 |
| 营业额 | 模拟周期内订单金额总和 |
| 客单价 | 平均每单消费金额 |
| 平均评分 | 用户评价平均分 |
| 平均配送时长 | 订单平均配送时间 |
| 差评率 | 负面评价占全部评价的比例 |

该模块用于模拟商家日常查看经营状态的首页场景。

---

## 2. 差评归因分析

差评归因分析基于 `mock_reviews.csv` 中的评价文本和问题类型字段，展示：

- 评价问题类型分布
- 负面评价列表
- 高频问题总结
- AI 差评归因结论示例

该模块用于展示 MealMind AI 如何帮助商家从大量评价中快速识别主要问题。

---

## 3. 营销活动分析

营销活动分析基于 `mock_marketing.csv` 中的活动数据，展示：

- 活动类型
- 目标用户群体
- 活动预期效果
- 实际订单提升
- ROI 对比
- AI 营销建议示例

该模块用于模拟商家查看营销活动效果，并获得下一步运营建议的场景。

---

## 4. AI 经营诊断示例

AI 经营诊断示例用于展示商家自然语言提问后的产品交互流程。

示例问题：

```text
为什么我这周订单下降了？
```

系统输出内容包括：

1. 异常现象
2. 关键证据
3. 可能原因
4. 优化建议
5. 客服回复示例

当前 Demo 中的诊断内容为规则化示例，后续可以接入大语言模型 API，实现动态生成。

---

## 六、当前 Demo 的限制

| 限制 | 说明 |
|---|---|
| 使用模拟数据 | 当前没有接入真实外卖平台数据 |
| 诊断逻辑较简单 | 当前主要用于展示产品流程 |
| 未接入真实大模型 | AI 诊断内容为预设示例 |
| 未实现自动执行 | 不支持自动发券、自动退款或自动修改活动 |
| 未做用户权限系统 | 当前 Demo 仅展示单一商家视角 |
