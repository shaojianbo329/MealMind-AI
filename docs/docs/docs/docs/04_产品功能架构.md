# 04 产品功能架构

## 一、产品设计目标

MealMind AI 是一款面向外卖商家的智能经营分析与运营决策助手。

产品的核心目标不是再做一个商家后台，而是在已有订单、评价、营销和客服数据的基础上，帮助商家完成从经营异常发现到策略执行辅助的闭环。

核心链路如下：

```text
发现经营异常
↓
分析异常原因
↓
生成优化策略
↓
辅助执行动作
↓
追踪执行效果
```

因此，MealMind AI 的功能架构应围绕“经营诊断闭环”展开，而不是简单堆叠 AI 功能。

---

## 二、产品业务闭环

| 业务环节 | 对应功能模块 | 解决的问题 |
|---|---|---|
| 数据输入 | 数据接入层 | 获取订单、评价、营销、客服等基础数据 |
| 异常识别 | 经营看板 | 帮助商家发现订单下降、差评上升等问题 |
| 归因分析 | AI 经营诊断 | 分析问题产生的原因 |
| 策略生成 | 营销策略生成 | 给出可执行的运营建议 |
| 执行辅助 | 客服回复辅助 | 帮助商家快速回复评价和投诉 |
| 效果追踪 | 效果复盘 | 判断建议是否带来改善 |

---

## 三、整体功能架构

MealMind AI 的功能架构分为 6 个一级模块：

| 一级模块 | 模块定位 | 优先级 |
|---|---|---|
| 数据接入层 | 接入订单、评价、营销、客服等数据 | P0 |
| 经营看板 | 展示关键指标并识别异常 | P0 |
| AI 经营诊断 | 分析经营异常原因并生成诊断报告 | P0 |
| 评价洞察分析 | 对用户评价进行分类、归因和趋势分析 | P0 |
| 营销策略生成 | 生成优惠券、套餐、爆品和复购策略 | P1 |
| 客服回复辅助 | 生成评价回复、投诉安抚和补偿建议 | P1 |

## 五、核心功能模块说明

### 1. 数据接入层

数据接入层为 AI 诊断和策略生成提供基础数据来源。

| 数据类型 | 示例字段 | 作用 |
|---|---|---|
| 订单数据 | 订单量、营业额、客单价、下单时间 | 判断经营趋势 |
| 流量数据 | 曝光量、访问量、转化率 | 判断流量和转化问题 |
| 评价数据 | 评分、评价文本、评价时间 | 判断用户满意度 |
| 菜品数据 | 菜品名称、销量、价格、退款率 | 识别问题菜品和爆品 |
| 营销数据 | 活动类型、优惠金额、活动周期 | 评估营销效果 |
| 客服数据 | 投诉内容、处理状态、补偿方式 | 辅助服务优化 |

MVP 阶段不需要真实接入平台后台，可以通过模拟数据验证产品逻辑。

---

### 2. 经营看板

经营看板用于帮助商家快速了解门店当前经营状态，并发现异常指标。

| 指标 | 含义 | 作用 |
|---|---|---|
| 订单量 | 一定时间内完成的订单数量 | 判断经营规模变化 |
| 营业额 | 一定时间内总销售收入 | 判断收入水平 |
| 客单价 | 平均每单消费金额 | 判断用户消费水平 |
| 曝光量 | 店铺被用户看到的次数 | 判断流量入口表现 |
| 下单转化率 | 下单用户 / 访问用户 | 判断商品和价格吸引力 |
| 复购率 | 重复购买用户占比 | 判断用户忠诚度 |
| 差评率 | 差评数量 / 总评价数量 | 判断产品和服务体验 |

页面设计要点：

- 首页展示核心指标卡片。
- 支持今日、近 7 天、近 30 天切换。
- 支持与上周期、同行均值对比。
- 对异常指标进行醒目标记。
- 点击异常指标后进入 AI 经营诊断页。

---

### 3. AI 经营诊断

AI 经营诊断用于帮助商家回答“为什么经营数据变差了”，并输出有证据支撑的诊断报告。

| 商家问题 | 系统分析方向 |
|---|---|
| 为什么这周订单下降了？ | 曝光量、访问量、转化率、差评率、活动变化 |
| 为什么差评变多了？ | 评价文本、投诉内容、配送时效、问题菜品 |
| 为什么活动效果不好？ | 活动前后订单、新客、复购、利润变化 |
| 为什么转化率下降了？ | 菜品价格、图片、评分、配送费、竞品活动 |

诊断报告包括：

1. 异常现象说明
2. 关键数据变化
3. 可能原因排序
4. 支撑证据
5. 可执行优化建议
6. 建议优先级
7. 后续追踪指标

示例输出：

```text
本周订单量较上周下降 18%。主要原因可能是曝光量下降和差评率上升。

关键证据：
1. 曝光量下降 12%，说明进入店铺的潜在用户减少。
2. 下单转化率下降 6%，说明用户进店后下单意愿降低。
3. 近 7 天“包装洒漏”相关差评明显增加，占差评总量的 42%。

建议：
1. 优先优化汤类菜品包装。
2. 对近 30 天购买过汤类菜品的用户发放复购券。
3. 在商品页增加“汤粉分装”“密封包装”说明。
```

---

### 4. 评价洞察分析

评价洞察分析用于将分散的用户评价转化为结构化洞察，帮助商家快速理解用户不满来源。

| 分类 | 示例关键词 | 可能动作 |
|---|---|---|
| 口味问题 | 太咸、太淡、不新鲜、味道变了 | 调整菜品配方 |
| 包装问题 | 洒漏、破损、汤撒了、包装差 | 更换包装方案 |
| 配送问题 | 配送慢、送错、骑手态度差 | 优化出餐和配送衔接 |
| 价格问题 | 太贵、分量少、不划算 | 调整套餐或优惠 |
| 服务问题 | 回复慢、处理不好、不负责 | 优化客服处理流程 |

输出内容包括：

- 好评 / 中评 / 差评占比
- 高频负面关键词
- 差评原因分类
- 问题菜品识别
- 用户情绪变化
- 可执行优化建议

---

### 5. 营销策略生成

营销策略生成用于帮助商家根据经营目标生成更合理的营销策略，避免盲目降价或无效满减。

| 策略类型 | 使用场景 | 目标 |
|---|---|---|
| 新客券 | 新用户占比低 | 拉新 |
| 复购券 | 老客复购下降 | 召回老客 |
| 满减活动 | 客单价偏低 | 提高下单金额 |
| 套餐组合 | 单品销量分散 | 提升组合销售 |
| 爆品推荐 | 缺少高转化菜品 | 提升转化率 |

AI 需要给出：

1. 推荐策略
2. 推荐原因
3. 适用用户群体
4. 预期效果
5. 风险提示
6. 后续观察指标

---

### 6. 客服回复辅助

客服回复辅助用于帮助商家快速、规范、低成本地回复用户评价和投诉。

| 类型 | 说明 |
|---|---|
| 好评回复 | 增强用户好感和品牌形象 |
| 普通差评回复 | 表达歉意并说明改进措施 |
| 严重投诉回复 | 安抚用户并引导私信处理 |
| 补偿建议 | 根据问题严重程度建议优惠券或退款 |
| 多语气版本 | 标准版、温和版、品牌版、诚恳版 |

示例：

用户评价：

```text
汤全洒了，包装太差了，等了很久才送到。
```

AI 回复建议：

```text
非常抱歉给您带来了不好的用餐体验。关于您反馈的汤品洒漏和等待时间较长问题，我们已经记录并会立即优化汤类菜品的密封包装，同时加强高峰期出餐和配送衔接。感谢您的反馈，也希望后续能给您带来更好的体验。
```

## 七、MVP 版本范围

| MVP 功能 | 说明 |
|---|---|
| 模拟数据接入 | 使用 mock_orders、mock_reviews、mock_marketing 等数据 |
| 经营看板 | 展示核心经营指标 |
| 订单下降诊断 | 分析订单下降原因 |
| 差评归因分析 | 对近期差评进行分类 |
| AI 经营建议 | 输出可执行建议 |
| 客服回复生成 | 根据评价生成回复话术 |

MVP 暂不包含自动投放营销活动和真实平台后台对接，而是通过模拟订单数据和评价数据验证产品逻辑。

---

## 八、小结

MealMind AI 的产品功能架构不是简单堆叠 AI 能力，而是围绕外卖商家的经营诊断闭环展开。

其核心逻辑是：

```text
数据接入
↓
异常识别
↓
原因分析
↓
策略生成
↓
执行辅助
↓
效果追踪
```

通过这一闭环，MealMind AI 可以帮助中小外卖商家降低数据分析门槛，提高经营问题定位效率，并将 AI 能力转化为真实的运营决策支持。
