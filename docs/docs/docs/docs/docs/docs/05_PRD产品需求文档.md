## 二、产品背景

外卖商家在日常经营中需要同时关注订单量、营业额、客单价、曝光量、转化率、复购率、差评率、营销活动效果等多个指标。

对于大型连锁商家来说，通常有专业运营团队进行数据分析和策略制定。但对于大量中小商家而言，经营决策更多依赖经验判断。当订单下降、差评增加或活动效果不佳时，商家往往难以快速判断原因，也难以生成具体可执行的优化方案。

MealMind AI 希望通过大语言模型和多 Agent 工作流，将订单数据、用户评价、营销活动和客服反馈整合起来，帮助商家完成从问题发现、原因分析到策略生成的经营辅助闭环。

---

## 三、产品目标

### 1. 用户目标

帮助外卖商家更快理解经营数据，降低数据分析门槛，并获得可执行的运营建议。

### 2. 业务目标

提升商家经营效率，降低商家流失风险，提高平台商家服务能力和用户体验。

### 3. 产品目标

构建一个面向外卖商家的 AI 经营诊断助手，优先解决以下问题：

- 商家看不懂经营数据
- 商家不知道订单下降原因
- 商家难以从差评中提炼问题
- 商家不会制定有效营销策略
- 商家回复评价和投诉效率低

---

## 四、目标用户

| 用户类型 | 用户特征 | 核心需求 |
|---|---|---|
| 中小外卖商家 | 缺少专业运营人员，依赖经验经营 | 快速发现问题并获得建议 |
| 门店运营人员 | 每天查看订单、评价和活动数据 | 提高分析和执行效率 |
| 连锁品牌区域经理 | 管理多家门店，关注异常门店 | 快速识别门店差异和共性问题 |
| 平台商家运营人员 | 负责商家扶持和经营指导 | 提高商家服务效率 |

## 五、核心用户场景

## 场景一：订单突然下降

### 用户问题

> 为什么我这周订单下降了？

### 触发条件

商家发现近 7 天订单量明显低于上周或低于历史平均水平。

### 用户痛点

商家只能看到订单下降，但不知道是曝光下降、转化下降、差评增加、价格变化、活动减少还是竞品影响。

### 产品解决方案

系统自动读取订单、流量、评价和营销数据，通过 AI 经营诊断生成原因分析和优化建议。

---

## 场景二：差评突然增加

### 用户问题

> 最近为什么差评变多了？

### 触发条件

近 7 天差评率高于上周期，或评分出现明显下降。

### 用户痛点

差评文本分散，商家人工查看效率低，难以判断主要问题集中在哪些方面。

### 产品解决方案

系统对评价进行情绪识别、关键词提取和原因分类，输出差评归因结果和处理建议。

---

## 场景三：营销活动效果不好

### 用户问题

> 这次满减活动为什么没有带来更多订单？

### 触发条件

商家设置营销活动后，订单量、客单价、新客占比或利润未达到预期。

### 用户痛点

商家不知道活动是否真正有效，也不知道下一轮应该调整满减、套餐还是优惠券。

### 产品解决方案

系统对比活动前后数据，分析活动对订单、新客、复购和客单价的影响，并生成下一轮营销建议。

---

## 场景四：需要回复差评或投诉

### 用户问题

> 这条差评怎么回复比较好？

### 触发条件

商家收到差评、投诉或负面反馈。

### 用户痛点

商家回复评价耗时长，且容易出现语气生硬、解释不足或处理不规范的问题。

### 产品解决方案

系统识别评价问题类型和用户情绪，生成不同语气版本的回复建议，并提供补偿或处理建议。

## 七、核心功能需求

# 功能一：经营看板

## 1. 功能目标

帮助商家快速了解门店经营状态，并发现异常指标。

## 2. 功能入口

产品首页。

## 3. 页面内容

| 模块 | 内容 |
|---|---|
| 核心指标区 | 订单量、营业额、客单价、曝光量、转化率、复购率、差评率 |
| 趋势对比区 | 今日、近 7 天、近 30 天数据变化 |
| 异常提醒区 | 展示异常指标和变化幅度 |
| AI 快捷提问区 | 支持商家直接提问经营问题 |

## 4. 交互逻辑

1. 用户进入首页。
2. 系统展示当前门店经营指标。
3. 异常指标以醒目标签提示。
4. 用户点击异常指标。
5. 系统跳转至 AI 经营诊断页。

## 5. 异常规则示例

| 异常类型 | 判断规则 |
|---|---|
| 订单下降 | 近 7 天订单量较上周期下降超过 15% |
| 差评上升 | 近 7 天差评率较上周期上升超过 20% |
| 转化下降 | 下单转化率较上周期下降超过 10% |
| 客单价异常 | 客单价较历史均值波动超过 15% |

---

# 功能二：AI 经营诊断

## 1. 功能目标

帮助商家分析经营异常原因，并生成有证据支撑的诊断报告。

## 2. 功能入口

- 经营看板异常指标点击进入
- 首页 AI 提问框进入
- 诊断报告历史记录进入

## 3. 输入内容

| 输入类型 | 示例 |
|---|---|
| 商家自然语言问题 | 为什么我这周订单下降了？ |
| 经营指标数据 | 订单量、曝光量、转化率、客单价、差评率 |
| 评价数据 | 评分、评价文本、关键词 |
| 营销数据 | 活动类型、优惠力度、活动时间 |
| 菜品数据 | 菜品销量、价格、退款率 |

## 4. 输出内容

AI 诊断报告包括：

1. 异常现象说明
2. 关键数据变化
3. 可能原因排序
4. 支撑证据
5. 优化建议
6. 建议优先级
7. 后续观察指标

## 5. 输出示例

```text
本周订单量较上周下降 18%。

主要原因可能是曝光量下降和差评率上升共同导致。

关键证据：
1. 曝光量下降 12%，说明进入店铺的潜在用户减少。
2. 下单转化率下降 6%，说明用户进店后下单意愿降低。
3. 近 7 天“包装洒漏”相关差评占差评总量的 42%。

建议：
1. 优先优化汤类菜品包装。
2. 对近 30 天购买过汤类菜品的用户发放复购券。
3. 在商品页增加“汤粉分装”“密封包装”说明。

后续观察指标：
订单量、转化率、包装类差评占比、复购券使用率。
```

## 6. 异常情况

| 异常情况 | 处理方式 |
|---|---|
| 数据不足 | 提示用户当前数据不足，仅输出初步判断 |
| 评价文本过少 | 提示评价样本不足，不进行强结论归因 |
| 指标变化不明显 | 输出“暂无明显异常”，并展示可关注指标 |
| AI 生成失败 | 提示重新生成，并保留原始数据展示 |

# 功能三：差评归因分析

## 1. 功能目标

帮助商家从大量用户评价中快速识别主要问题类型。

## 2. 功能入口

- 评价分析页
- AI 经营诊断报告中的差评模块
- 差评详情页

## 3. 分类维度

| 问题类型 | 示例关键词 | 对应建议 |
|---|---|---|
| 口味问题 | 太咸、太淡、不新鲜、味道变了 | 调整菜品配方 |
| 包装问题 | 洒漏、破损、汤撒了 | 更换包装方案 |
| 配送问题 | 配送慢、送错、凉了 | 优化出餐和配送衔接 |
| 价格问题 | 太贵、分量少、不划算 | 调整套餐或优惠 |
| 服务问题 | 回复慢、处理不好 | 优化客服流程 |

## 4. 输出内容

| 输出项 | 说明 |
|---|---|
| 差评分类占比 | 展示各类问题占比 |
| 高频关键词 | 提取用户集中反馈的问题 |
| 问题菜品 | 识别被多次提及的菜品 |
| 代表性评价 | 展示典型用户原话 |
| 优化建议 | 根据问题类型生成具体动作 |

## 5. 示例输出

```text
近 7 天差评主要集中在包装问题和配送问题。

分类结果：
- 包装问题：42%
- 配送问题：28%
- 口味问题：16%
- 价格问题：9%
- 服务问题：5%

代表性评价：
“汤洒了一半，包装真的不行。”
“等了很久才送到，饭都凉了。”

建议：
1. 优先更换汤类菜品密封包装。
2. 在高峰期适当延长预计送达时间。
3. 对包装类差评用户发放小额补偿券。
```

# 功能四：营销策略生成

## 1. 功能目标

根据经营数据和商家目标，生成更合理的营销策略。

## 2. 功能入口

- 营销建议页
- AI 经营诊断报告
- 活动复盘页

## 3. 商家可选择目标

| 目标 | 推荐策略方向 |
|---|---|
| 提升订单量 | 新客券、限时折扣、爆品引流 |
| 提升客单价 | 满减、套餐组合、加价购 |
| 提升复购率 | 老客券、会员券、复购提醒 |
| 降低差评影响 | 补偿券、服务恢复策略 |
| 打造爆品 | 单品折扣、套餐组合、首页推荐 |

## 4. 输出内容

AI 需要输出：

1. 推荐活动类型
2. 推荐理由
3. 适用用户群体
4. 预期提升指标
5. 风险提示
6. 后续观察指标

## 5. 示例输出

```text
推荐策略：老客复购券 + 汤类套餐组合

推荐原因：
近 30 天老客复购率下降 9%，同时汤类菜品因包装问题导致差评增加。建议在优化包装后，对购买过汤类菜品的老客发放 5 元复购券。

适用用户：
近 30 天购买过汤类菜品，且最近 14 天未复购的用户。

预期目标：
提升老客复购率，降低包装问题带来的负面影响。

风险提示：
优惠力度不宜过高，避免牺牲过多利润。
```

# 功能五：客服回复生成

## 1. 功能目标

帮助商家快速生成规范、礼貌、可执行的用户评价回复。

## 2. 功能入口

- 评价详情页
- 投诉处理页
- AI 诊断报告中的评价模块

## 3. 输入内容

| 输入项 | 示例 |
|---|---|
| 用户评价 | 汤全洒了，包装太差了，等了很久才送到 |
| 评价星级 | 1 星 |
| 问题类型 | 包装问题、配送问题 |
| 商家语气偏好 | 标准、温和、诚恳、品牌化 |
| 是否需要补偿 | 是 / 否 |

## 4. 输出内容

系统生成不同版本回复：

| 版本 | 说明 |
|---|---|
| 标准版 | 礼貌、简洁、适合大多数场景 |
| 诚恳版 | 更强调道歉和改进 |
| 品牌版 | 适合连锁品牌统一口径 |
| 补偿引导版 | 引导用户私信或领取补偿 |

## 5. 示例输出

```text
非常抱歉给您带来了不好的用餐体验。关于您反馈的汤品洒漏和等待时间较长问题，我们已经记录并会立即优化汤类菜品的密封包装，同时加强高峰期出餐和配送衔接。感谢您的反馈，也希望后续能给您带来更好的体验。
```

## 十、数据指标

## 1. 产品使用指标

| 指标 | 含义 |
|---|---|
| 诊断功能点击率 | 商家是否愿意使用 AI 诊断 |
| 诊断报告生成成功率 | AI 功能是否稳定可用 |
| 诊断报告阅读完成率 | 商家是否认真查看结果 |
| 建议采纳率 | 商家是否执行 AI 建议 |
| 客服回复使用率 | 商家是否复制或使用 AI 回复 |

## 2. 业务效果指标

| 指标 | 含义 |
|---|---|
| 订单量变化 | 判断经营建议是否带来订单改善 |
| 转化率变化 | 判断商品和活动优化是否有效 |
| 差评率变化 | 判断服务和体验问题是否改善 |
| 复购率变化 | 判断老客召回策略是否有效 |
| 活动 ROI | 判断营销策略是否具备投入产出价值 |

## 3. AI 质量指标

| 指标 | 含义 |
|---|---|
| 归因准确率 | AI 判断问题原因是否合理 |
| 建议可执行性评分 | 商家是否认为建议可落地 |
| 回复满意度 | 商家对客服回复质量的评价 |
| 幻觉率 | AI 是否输出无依据或错误建议 |
| 人工修改率 | 商家使用回复前需要修改的比例 |

## 十二、风险与限制

| 风险 | 说明 | 应对方式 |
|---|---|---|
| 数据真实性不足 | 个人项目无法接入真实平台数据 | 使用模拟数据，并说明数据字段和逻辑 |
| AI 输出泛化 | 大模型可能给出空泛建议 | 强制要求输出证据和指标来源 |
| 建议不可执行 | AI 建议可能超出商家能力 | 限制建议为商家可操作动作 |
| 平台规则不准确 | 不同平台规则可能变化 | 使用 RAG 接入规则知识库 |
| 自动执行风险 | 自动投放活动或补偿涉及资金风险 | MVP 阶段只做建议，不做自动执行 |

## 十三、小结

MealMind AI 的 MVP 重点不是做一个完整商家后台，而是验证 AI 在外卖商家经营诊断中的产品价值。

PRD 的核心逻辑可以总结为：

```text
商家提出经营问题
↓
系统读取订单、评价、营销等数据
↓
AI 多维归因分析
↓
生成可解释、可执行的建议
↓
商家执行
↓
系统追踪效果
```

通过这一闭环，MealMind AI 可以体现 AI 产品经理所需要的场景理解、需求分析、功能设计、AI 能力拆解和指标思维。
