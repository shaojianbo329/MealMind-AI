# 06 Agent 工作流设计

## 一、设计目标

MealMind AI 的核心亮点不是简单调用大模型生成建议，而是通过多 Agent 工作流，将外卖商家的经营问题拆解为多个可分析、可协作、可追踪的任务。

在外卖经营场景中，商家提出的问题往往不是单一问题。例如：

> 为什么我这周订单下降了？

这个问题背后可能涉及订单量、曝光量、转化率、差评率、营销活动、菜品价格、配送体验等多个因素。

因此，MealMind AI 需要通过多个 Agent 分工协作，完成从数据分析、评价洞察、营销建议到客服回复的完整流程。

---

## 二、Agent 总体架构

MealMind AI 设计 4 个核心 Agent：

| Agent 名称 | 核心职责 | 输入数据 | 输出结果 |
|---|---|---|---|
| 数据分析 Agent | 分析订单、流量、转化、客单价等经营指标 | 订单数据、流量数据、菜品数据 | 经营异常判断、关键指标变化 |
| 评价洞察 Agent | 分析用户评价、投诉和差评原因 | 评价数据、投诉数据 | 差评分类、关键词、问题菜品 |
| 营销策略 Agent | 根据经营目标生成营销和运营建议 | 经营指标、评价结果、菜品数据 | 优惠券、套餐、爆品、复购策略 |
| 客服话术 Agent | 生成评价回复和投诉处理话术 | 用户评价、问题类型、商家语气偏好 | 回复文案、补偿建议、处理建议 |

---

## 三、Agent 协同流程

当商家提出经营问题后，系统会按照以下流程进行处理：

```text
商家输入问题
↓
问题识别与任务拆解
↓
数据分析 Agent 分析经营指标
↓
评价洞察 Agent 分析用户反馈
↓
营销策略 Agent 生成运营建议
↓
客服话术 Agent 生成回复和安抚话术
↓
汇总生成经营诊断报告
↓
商家选择执行动作
↓
系统追踪后续效果
```

## 四、典型场景工作流

## 场景一：订单下降诊断

### 1. 商家输入

```text
为什么我这周订单下降了？
```

### 2. 系统任务拆解

系统将该问题拆解为以下任务：

| 子任务 | 负责 Agent |
|---|---|
| 判断订单是否真的下降 | 数据分析 Agent |
| 分析曝光量、转化率、客单价变化 | 数据分析 Agent |
| 检查近期差评和投诉变化 | 评价洞察 Agent |
| 判断是否与营销活动变化有关 | 营销策略 Agent |
| 生成优化建议 | 营销策略 Agent |
| 生成相关差评回复模板 | 客服话术 Agent |

### 3. Agent 执行过程

```text
数据分析 Agent
↓
发现近 7 天订单量较上周下降 18%，曝光量下降 12%，转化率下降 6%

评价洞察 Agent
↓
发现近 7 天包装类差评明显增加，占差评总量 42%

营销策略 Agent
↓
判断订单下降与曝光下降、差评上升和复购不足有关，建议优化包装并发放老客复购券

客服话术 Agent
↓
生成针对包装洒漏、配送慢等差评的回复话术
```

### 4. 最终输出

```text
本周订单量较上周下降 18%。主要原因可能是曝光量下降、转化率下降和包装类差评上升共同导致。

关键证据：
1. 曝光量下降 12%，说明进入店铺的潜在用户减少。
2. 下单转化率下降 6%，说明用户进店后下单意愿降低。
3. 包装类差评占近期差评的 42%，主要集中在汤类菜品。

建议：
1. 优先优化汤类菜品密封包装。
2. 在商品页增加“汤粉分装”“密封包装”说明。
3. 对近 30 天购买过汤类菜品的老客发放 5 元复购券。
4. 针对包装类差评使用 AI 生成的诚恳回复话术。

后续观察指标：
订单量、转化率、包装类差评占比、复购券使用率。
```

## 场景二：差评归因分析

### 1. 商家输入

```text
最近为什么差评变多了？
```

### 2. 系统任务拆解

| 子任务 | 负责 Agent |
|---|---|
| 判断差评是否显著增加 | 数据分析 Agent |
| 对近期评价进行情绪识别 | 评价洞察 Agent |
| 提取高频负面关键词 | 评价洞察 Agent |
| 识别问题菜品 | 评价洞察 Agent |
| 生成服务优化建议 | 营销策略 Agent |
| 生成差评回复话术 | 客服话术 Agent |

### 3. 最终输出

```text
近 7 天差评率较上周上升 23%。差评主要集中在包装问题和配送问题。

差评分类：
- 包装问题：42%
- 配送问题：28%
- 口味问题：16%
- 价格问题：9%
- 服务问题：5%

高频关键词：
“洒漏”“包装差”“送到凉了”“配送慢”

问题菜品：
酸辣粉、牛肉汤饭、番茄肥牛汤

建议：
1. 优先更换汤类菜品包装。
2. 对汤类菜品增加“汤粉分装”说明。
3. 高峰期适当延长预计送达时间，降低用户预期落差。
4. 对包装类差评用户发放小额补偿券。
```

## 五、各 Agent 详细设计

## 1. 数据分析 Agent

### 角色定位

数据分析 Agent 负责从订单、流量、转化、客单价、复购率等数据中发现经营异常。

### 输入数据

| 数据类型 | 示例字段 |
|---|---|
| 订单数据 | order_id、order_time、order_amount、user_id |
| 流量数据 | exposure、visit、conversion_rate |
| 菜品数据 | item_name、item_sales、item_price |
| 营销数据 | campaign_type、discount_amount、campaign_period |

### 输出内容

| 输出项 | 说明 |
|---|---|
| 异常指标 | 哪些指标发生明显变化 |
| 变化幅度 | 与上周期或历史均值相比的变化 |
| 初步判断 | 异常可能与哪些因素相关 |
| 需进一步分析项 | 是否需要评价或营销 Agent 继续分析 |

### Prompt 示例

```text
你是 MealMind AI 中的数据分析 Agent。请根据商家的订单、流量、转化率、客单价和营销数据，判断当前经营是否存在异常。

你需要输出：
1. 异常指标
2. 变化幅度
3. 可能原因
4. 需要其他 Agent 继续分析的方向

要求：
- 不要输出没有数据支持的结论
- 每个判断都要说明依据
- 建议使用商家能理解的语言
```

---

## 2. 评价洞察 Agent

### 角色定位

评价洞察 Agent 负责分析用户评价、差评、投诉内容和情绪变化，帮助商家理解用户不满来源。

### 输入数据

| 数据类型 | 示例字段 |
|---|---|
| 评价数据 | rating、review_text、review_time |
| 投诉数据 | complaint_type、complaint_content |
| 菜品数据 | item_name、item_category |
| 订单数据 | order_time、delivery_time |

### 输出内容

| 输出项 | 说明 |
|---|---|
| 情绪分类 | 好评、中评、差评 |
| 问题类型 | 口味、包装、配送、价格、服务 |
| 高频关键词 | 用户集中提到的问题 |
| 问题菜品 | 被多次提及的菜品 |
| 代表性评价 | 典型用户反馈原文 |
| 优化建议 | 对应问题的处理动作 |

### Prompt 示例

```text
你是 MealMind AI 中的评价洞察 Agent。请分析以下用户评价，并对评价进行分类和归因。

你需要输出：
1. 好评、中评、差评比例
2. 差评原因分类
3. 高频负面关键词
4. 被多次提及的问题菜品
5. 代表性用户评价
6. 可执行优化建议

分类维度：
- 口味问题
- 包装问题
- 配送问题
- 价格问题
- 服务问题

要求：
- 不要只总结，要给出可执行建议
- 如果样本量不足，需要明确提示
```

---

## 3. 营销策略 Agent

### 角色定位

营销策略 Agent 负责根据经营目标和数据变化，生成优惠券、套餐组合、爆品推荐和复购策略。

### 输入数据

| 数据类型 | 示例字段 |
|---|---|
| 经营指标 | 订单量、客单价、转化率、复购率 |
| 菜品数据 | 菜品销量、毛利、价格、退款率 |
| 用户数据 | 新客占比、老客复购、用户偏好 |
| 营销数据 | 活动类型、优惠力度、活动效果 |

### 输出内容

| 输出项 | 说明 |
|---|---|
| 推荐策略 | 新客券、复购券、满减、套餐、爆品 |
| 推荐原因 | 为什么推荐该策略 |
| 适用用户 | 新客、老客、沉默用户、高价值用户 |
| 预期影响 | 预计改善哪些指标 |
| 风险提示 | 是否可能影响利润或评分 |
| 观察指标 | 后续应追踪哪些数据 |

### Prompt 示例

```text
你是 MealMind AI 中的营销策略 Agent。请根据商家的经营数据、用户评价和菜品表现，生成可执行的营销策略。

你需要输出：
1. 推荐策略
2. 推荐原因
3. 适用用户群体
4. 预期提升指标
5. 风险提示
6. 后续观察指标

要求：
- 不要只说“建议做优惠活动”
- 必须说明推荐哪种活动、面向谁、为什么
- 需要考虑利润风险和用户体验
```

---

## 4. 客服话术 Agent

### 角色定位

客服话术 Agent 负责根据用户评价、投诉内容和问题类型，生成符合平台规范和商家语气的回复话术。

### 输入数据

| 数据类型 | 示例字段 |
|---|---|
| 用户评价 | review_text、rating |
| 问题类型 | 包装问题、配送问题、口味问题 |
| 商家语气 | 标准、诚恳、温和、品牌化 |
| 处理方式 | 道歉、解释、补偿、引导私信 |

### 输出内容

| 输出项 | 说明 |
|---|---|
| 标准版回复 | 适合大多数评价场景 |
| 诚恳版回复 | 更强调道歉和改进 |
| 品牌版回复 | 适合连锁品牌统一口径 |
| 补偿引导版回复 | 引导用户进一步沟通或补偿 |
| 注意事项 | 提醒商家避免承诺过度或用语不当 |

### Prompt 示例

```text
你是 MealMind AI 中的客服话术 Agent。请根据用户评价生成商家回复话术。

用户评价：
{{review_text}}

问题类型：
{{issue_type}}

商家语气：
{{tone}}

你需要输出：
1. 标准版回复
2. 诚恳版回复
3. 补偿引导版回复
4. 回复注意事项

要求：
- 语气礼貌、真诚
- 不推卸责任
- 不承诺平台无法保证的事项
- 回复应适合商家直接复制使用
```

## 六、Agent 协作机制

## 1. 任务拆解机制

用户输入问题后，系统先判断问题类型：

| 用户问题类型 | 触发 Agent |
|---|---|
| 订单下降、转化下降、客单价异常 | 数据分析 Agent |
| 差评增加、评分下降、投诉变多 | 评价洞察 Agent |
| 活动效果差、想提升订单 | 营销策略 Agent |
| 需要回复评价或投诉 | 客服话术 Agent |
| 复杂经营问题 | 多 Agent 协同 |

---

## 2. 信息传递机制

不同 Agent 之间需要传递中间结果。

```text
数据分析 Agent
输出：订单下降 18%，转化率下降 6%
↓
评价洞察 Agent
输出：包装类差评占比 42%
↓
营销策略 Agent
输出：优化包装 + 老客复购券
↓
客服话术 Agent
输出：包装类差评回复模板
```

---

## 3. 决策汇总机制

最终报告不直接拼接各 Agent 的结果，而是进行统一汇总，形成商家能理解的报告结构：

```text
1. 发生了什么
2. 可能为什么发生
3. 有哪些数据证据
4. 应该先做什么
5. 后续看哪些指标
```

---

## 七、AI 风险控制

| 风险 | 表现 | 控制方式 |
|---|---|---|
| 幻觉风险 | AI 编造不存在的数据或原因 | 要求所有结论必须引用输入数据 |
| 建议空泛 | 输出“提升服务质量”等泛泛建议 | 强制输出具体动作和适用场景 |
| 自动执行风险 | AI 自动发券或退款造成损失 | MVP 阶段只给建议，不自动执行 |
| 规则错误 | 违反平台营销或客服规则 | 接入规则知识库或人工审核 |
| 归因错误 | 错把相关性当因果 | 输出“可能原因”，并标注证据强度 |

## 八、MVP 阶段实现方式

在个人作品集项目中，MVP 阶段不需要真实实现复杂 Agent 系统，可以通过以下方式展示产品逻辑：

| 模块 | 实现方式 |
|---|---|
| 数据输入 | 使用模拟订单、评价和营销数据 |
| Agent Prompt | 为每个 Agent 编写 Prompt 模板 |
| 输出样例 | 设计典型商家问题和 AI 输出结果 |
| 工作流图 | 用 Markdown 或流程图展示 Agent 协作 |
| Demo 页面 | 可选，用 Streamlit 展示简单交互 |

## 九、小结

MealMind AI 的 Agent 工作流设计体现了 AI 产品的核心价值：不是让大模型孤立地回答问题，而是围绕真实业务场景进行任务拆解、数据分析、评价洞察、策略生成和执行辅助。

其核心逻辑可以总结为：

```text
商家问题
↓
任务拆解
↓
多 Agent 分工
↓
证据化分析
↓
可执行建议
↓
效果追踪
```

通过这种设计，MealMind AI 能够更贴近外卖商家真实经营过程，也更符合 AI 产品经理岗位对场景理解、产品设计和 AI 能力拆解的要求。
