# 09 面试讲解稿

## 一、1 分钟版本

我做了一个面向外卖商家的 AI 经营助手项目，叫 MealMind AI。这个项目聚焦中小外卖商家看不懂经营数据、差评归因困难、营销活动依赖经验和客服回复效率低的问题。

我没有选择做一个完整商家后台，而是设计了一个商家后台之上的 AI 智能分析层。它基于订单、评价和营销数据，帮助商家完成经营诊断、差评归因、营销策略生成和客服回复辅助。

在 AI 设计上，我拆分了 4 个 Agent：数据分析 Agent、评价洞察 Agent、营销策略 Agent 和客服话术 Agent。项目中我完成了用户场景、竞品分析、PRD、功能架构、指标体系、Prompt 设计、Mock 数据和 Streamlit Demo。

这个项目的核心价值是把 AI 从通用问答变成围绕外卖经营问题的决策辅助工具。

## 二、3 分钟版本

### 1. 项目背景

外卖中小商家每天都会面对订单下降、差评增加、活动效果不好、客服回复慢等问题。但很多商家没有专业运营能力，虽然能看到后台数据，却不知道为什么变化、下一步该怎么做。

所以我设计了 MealMind AI，希望它作为商家后台上的 AI 分析层，把订单、评价和营销数据转化为商家能理解、能执行的经营建议。

### 2. 产品方案

产品主要有五个模块：

1. 经营看板：展示订单量、营业额、客单价、评分、差评率等指标；
2. AI 经营诊断：分析订单下降、转化下降、差评上升等问题；
3. 评价洞察分析：识别差评类型、高频关键词和问题菜品；
4. 营销策略生成：生成复购券、套餐、爆品等策略；
5. 客服回复辅助：生成差评回复和投诉安抚话术。

### 3. AI 设计

我设计了 4 个 Agent：

| Agent | 作用 |
|---|---|
| 数据分析 Agent | 判断经营指标是否异常 |
| 评价洞察 Agent | 分析差评和用户反馈 |
| 营销策略 Agent | 生成运营策略 |
| 客服话术 Agent | 生成回复话术 |

例如商家问“为什么我这周订单下降了”，系统会先分析订单和转化率，再分析差评和问题菜品，然后生成营销建议和客服回复，最后汇总成诊断报告。

### 4. 项目产出

我完成了项目背景、用户场景、竞品分析、功能架构、PRD、Agent 工作流、Prompt、Mock 数据、指标体系和 Streamlit Demo。

### 5. 项目价值

这个项目体现的是 AI 产品经理的完整思路：不是直接套一个大模型，而是从业务场景出发，明确用户痛点，拆解数据和 AI 能力，再设计可评估、可落地的产品闭环。
